Integrate the Ticketmaster Discovery API into PartyPulse to show real live events on the map alongside user-created events. Here is exactly what to do:

---

## SETUP

1. Install nothing new — use native fetch() for all API calls
2. The Ticketmaster API base URL is: https://app.ticketmaster.com/discovery/v2/
3. Store the API key in the .env file as: VITE_TICKETMASTER_API_KEY=your_key_here
4. Get a free API key from https://developer.ticketmaster.com (tell the user to add their own key)

---

## CREATE A NEW FILE: src/lib/ticketmaster.ts

This file should:
- Export a function called fetchTicketmasterEvents(lat: number, lng: number, radiusKm: number)
- Call the Ticketmaster API endpoint: /events.json
- Pass these params:
  - apikey: the env variable
  - latlong: `${lat},${lng}`
  - radius: radiusKm (convert to miles for the API: multiply km by 0.621)
  - unit: miles
  - size: 50
  - sort: date,asc
- Map the response to the same Event interface used in firestoreEvents.ts so they work everywhere in the app
- For each Ticketmaster event, map the fields like this:
  - id: "tm_" + event.id (prefix with tm_ so we know it's from Ticketmaster)
  - title: event.name
  - description: event.info || event.pleaseNote || ""
  - date: event.dates.start.localDate
  - time: event.dates.start.localTime || "00:00"
  - location.lat: event._embedded.venues[0].location.latitude (parse as float)
  - location.lng: event._embedded.venues[0].location.longitude (parse as float)
  - location.address: event._embedded.venues[0].name + ", " + event._embedded.venues[0].city.name
  - imageUrl: event.images[0]?.url || ""
  - going: [] (empty array as default)
  - creatorId: "ticketmaster"
  - creatorName: "Ticketmaster"
  - category: map the Ticketmaster segment/genre to the closest PartyPulse category:
      - Music → Club Night
      - Arts & Theatre → Social
      - Sports → Social
      - Family → Social
      - default → Party
  - isExternal: true (add this field to distinguish from user events)
  - externalUrl: event.url (link to buy tickets)

- Handle errors gracefully — if the API fails or returns no results, return an empty array silently
